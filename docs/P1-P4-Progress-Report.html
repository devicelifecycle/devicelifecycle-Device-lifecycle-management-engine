Universal Trade Quote Parser — 8 Layout Patterns

After analysing 27 real sheets across 2 customer workbooks (COE Corporate Trade Quotes & SCC ITAD Apple Trade Booklet), the parser handles every layout seen in the wild.

Gap 1
Header row not always row 0
Auto-detects by scoring rows 0–14 against keywords. Lambton → headers at row 10.
Gap 2
Multi-row merged column headers
"30 Days" group + "Good" sub → 30 Good. Handles Turow, ALSA.
Gap 3
Combined make+model+storage+color
"Apple iPhone 12 64GB Black" → brand + model + storage + color split automatically.
Gap 4
Missing Make column
Brand inferred from model: iPhone → Apple, Galaxy → Samsung.
Gap 5
Condition normalisation
50+ aliases: Functional→good, Defective→broken, Grade A→excellent, "minor scratch"→fair.
Gap 6
Pivot / transposed tables
Models as column headers, prices as rows — auto-detected and transposed. PAL Aero, AMA RFQ.
Gap 7
Storage-as-column-header
"32 GB" / "128 GB" are price columns — storage resolved from which column has a price.
Gap 8
Multi-sheet workbook picker
Sheet selector appears in import dialog when workbook has >1 sheet. Switching re-parses instantly.
